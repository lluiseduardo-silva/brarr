Regras

RuleSet JSON

Lista de regras serializada como JSON. Engine itera em ordem; cada regra que casa soma add_score e (opcionalmente) marca reject: true. Lista vazia {"rule":[]} = engine baseline.

Inserir exemplo:
Schema rápido 
{
  "rule": [
    {
      "name": "exemplo",
      "when": {
        "audio": "pt-br|pt-pt|pt|pt-any|en|jp|zh",
        "subtitle": "pt-br|pt-pt|pt-any|en|jp|zh",
        "hdr": true,
        "resolution": "min-720|min-1080|min-2160|exact-1080|exact-2160",
        "min_seeders": 5,
        "min_size_bytes": 1000000000,
        "max_size_bytes": 50000000000,
        "tracker": "capybara",
        "kind": "web-dl|bluray|remux|encode|hdtv|dvd",
        "video_codec": "x264|x265|av1",
        "release_group": "NeX",
        "proper": true,
        "repack": false,
        "title_contains": "dual",
        "title_matches": "(?i)\\bS\\d{2}E\\d{2}\\b",

        "all": [ {/* AND: todas casam */} ],
        "any": [ {/* OR: ao menos uma casa */} ],
        "not": {/* a subcondição NÃO casa */}
      },
      "add_score": 100,
      "tag": null,
      "reject": false
    }
  ]
}

Árvore booleana: um when casa quando todos os campos escalares presentes casam (AND) e os combinadores all/any/not casam. Aninhe à vontade. Faixa de resolução, por exemplo: {"all":[{"resolution":"min-1080"},{"not":{"resolution":"min-2160"}}]}.
Derivados do título: kind, video_codec, release_group, proper/repack saem da tokenização do nome (o codec é refinado pelo MediaInfo quando disponível) — best-effort.
Idiomas: pt-br/pt-pt/pt/pt-any, en, jp, zh.
